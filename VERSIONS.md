# V0.3 «SETTINGS»
Левая панель настроек с возможностью менять параметры в реальном времени:
- Скорость поворота (макс./мин., об/с), скорость вперёд и назад (макс./мин., м/с), дистанция следования (мин./макс., мм)
- Выбор модели YOLO (yolo11n / yolo26n) и устройства (CPU / GPU, с проверкой CUDA) — до подключения
- Управление углом гимбала (0–35°) — при подключении, вне трекинга
- Кнопка «Отключиться» на правой панели

Пересчёт скоростей переведён в физические единицы:
- Вращение: об/с платформы → RPM колёс через геометрию Mecanum (r=5 см, L=20 см)
- Движение: м/с → RPM колёс

Окно масштабируемое (`RESIZABLE`), при запуске открывается на весь экран. Минимальная ширина: LEFT_W + 480 + SIDEBAR_W. Видеопанель растягивается, боковые панели фиксированы по 320 px.

Фикс лога: длинные сообщения переносятся по словам, продолжение выровнено под метку времени.

# V0.2 «PYGAME»
Рефакторинг кода: единственный файл `main_window.py` разделён на независимые модули:
- `motion.py` — функции расчёта скоростей Mecanum-колёс
- `robot_controller.py` — обёртка над SDK DJI RoboMaster (подключение, камера, шасси)
- `tracker.py` — YOLO-детекция и BoT-SORT трекинг
- `app.py` — pygame-приложение (интерфейс + главный цикл)
- `main.py` — точка входа

Интерфейс переписан с tkinter + отдельного OpenCV-окна на единое pygame-окно:
- Видеопоток встроен в левую панель (960×720), масштабируется автоматически
- Правая боковая панель (320 px): индикаторы статуса, кнопки управления, цветной лог
- Поддержка WASD-управления непосредственно из pygame-окна (вне режима трекинга)
- Отображение расстояния, режима трекинга, времени поиска и FPS в реальном времени

# V0.1 «PROTO»
Создан пользовательский интерфейс. Управление трекингом осуществляется со стороны пользователя. На данном этапе реализовано управление линейной и угловой скоростями. Скорости смешения пока не задействуются. Для визуализации используется opencv. Для распознавания и трекинга YOLOv11 с кастомными настройками трекера.
