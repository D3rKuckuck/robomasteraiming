# Принцип работы системы слежения RoboMaster S1

## Общая архитектура

Система состоит из четырёх логических слоёв:

1. **Слой связи с роботом** — SDK DJI RoboMaster, подключение по Wi-Fi точке доступа (режим AP)
2. **Слой компьютерного зрения** — детекция людей через YOLOv11/YOLO26 + трекинг через BoT-SORT
3. **Слой управления движением** — пересчёт ошибки позиции и дистанции в RPM колёс
4. **Слой интерфейса** — левая панель настроек, видеопоток, правая панель с логом и кнопками

---

## Подключение к роботу

При нажатии кнопки «Подключиться» вызывается `_connect()` → `RobotController.connect(gimbal_pitch_deg)`:

- `ep_robot.initialize(conn_type="ap")` — подключение через Wi-Fi точку доступа робота
- Инициализируются подсистемы: `chassis`, `gimbal`, `sensor`
- Гимбал возвращается в центральное положение, затем наклоняется на угол, выбранный в панели настроек (по умолчанию 15°)
- Запускается подписка на TOF-дальномер: `sub_distance(freq=5, callback=_tof_callback)` — 5 Гц, данные в мм

Кнопка «Отключиться» корректно останавливает колёса, камеру, отписывается от датчика и закрывает SDK.

## Компьютерное зрения: детекция и трекинг

### Модель детекции

Пользователь выбирает модель до подключения:

| Модель | Файл весов | Рекомендуется для |
|---|---|---|
| YOLO11n | `yolo11n.pt` | GPU (CUDA) — быстрее |
| YOLO26n | `yolo26n.pt` | CPU — эффективнее |

При запуске трекинга модель вызывается на каждом кадре:

```python
results = model.track(frame, persist=True, tracker="custom_track.yaml",
                      classes=[0], conf=0.3, device=self.device)
```

- `classes=[0]` — только класс «человек»
- `conf=0.3` — порог уверенности детектора
- `persist=True` — трекер сохраняет состояние между кадрами
- `device` — `'cuda'` или `'cpu'`; при выборе GPU проверяется наличие CUDA через `torch.cuda.is_available()`

Если конфигурация (модель или устройство) изменилась между запусками трекинга, `PersonTracker` пересоздаётся.

### Конфигурация трекера (`custom_track.yaml`)

Используется алгоритм **BoT-SORT** (`tracker_type: botsort`).

Ключевые параметры:
| Параметр | Значение | Смысл |
|---|---|---|
| `track_high_thresh` | 0.25 | Порог первичного сопоставления детекций с треками |
| `new_track_thresh` | 0.8 | Порог для создания нового трека (снижает ложные треки) |
| `track_buffer` | 300 | Кадров хранить «потерянный» трек (долго = устойчиво к окклюзии) |
| `match_thresh` | 0.6 | Порог сходства при сопоставлении (IoU + внешний вид) |
| `gmc_method` | sparseOptFlow | Компенсация глобального движения камеры через оптический поток |
| `with_reid` | False | ReID-модель отключена (экономия вычислений) |

### Поток обработки кадра (`process_frame`)

1. Получить кадр с камеры: `ep_camera.read_cv2_image(strategy="newest")`
2. Запустить `model.track(...)` → получить боксы, ID треков, классы, уверенность
3. Для каждого человека с `conf > 0.3` — вычислить центр бокса `(center_x, center_y)`
4. Обновить словарь `self.tracks = {track_id: (cx, cy)}`
5. Нарисовать боксы (зелёный — не выбранная цель, красный — выбранная)

## Выбор цели

Цель выбирается кликом по видеообласти pygame-окна.
Находится ближайший трек в радиусе 100 пикселей от клика (в координатах кадра). Его ID записывается в `self.selected_id`.

## Управление движением

### Геометрия Mecanum-привода

Robomaster S1 оснащён 4 колёсами Mecanum. Используемые параметры:

| Параметр | Значение |
|---|---|
| Радиус колеса `r` | 0.05 м (диаметр 10 см) |
| Половина колёсной базы (продольная) `lx` | 0.10 м |
| Половина колёсной базы (поперечная) `ly` | 0.10 м |
| Суммарный рычаг `L = lx + ly` | 0.20 м |

Пересчёт физических единиц в RPM колёс:
- Вращение: `RPM = ω_rev_s × L × 60 / r = ω_rev_s × 240`
- Движение: `RPM = v_m_s × 60 / (2π × r) ≈ v_m_s × 191`

### Расчёт скоростей колёс (`calculate_movement_speeds`)

Функция принимает настраиваемые параметры (задаются в левой панели интерфейса):

| Параметр | Диапазон | Смысл |
|---|---|---|
| `max_rotation_rev_s` | 0.1–3.0 об/с | Макс. угловая скорость платформы |
| `min_rotation_rev_s` | 0.1–3.0 об/с | Мёртвая зона: ниже — вращение не подаётся |
| `max_fwd_m_s` | 0.1–8.0 м/с | Макс. скорость вперёд |
| `min_fwd_m_s` | 0.1–8.0 м/с | Мин. скорость при движении вперёд |
| `max_bwd_m_s` | 0.1–8.0 м/с | Макс. скорость назад |
| `min_bwd_m_s` | 0.1–8.0 м/с | Мин. скорость при движении назад |
| `min_dist_mm` | 200–4000 мм | Ближе — ехать назад |
| `max_dist_mm` | 500–6000 мм | Дальше — ехать на макс. скорости |

**Угловое воздействие:**
`error_x = target_x − frame_width / 2` масштабируется в `[−max_rotation_rev_s, +max_rotation_rev_s]`.
Если результат ниже `min_rotation_rev_s` — вращение = 0 (мёртвая зона).

**Линейное воздействие:**
Если `|error_x| > frame_width / 4` — движение заблокировано (сначала довернуть).
Иначе дистанция по TOF определяет направление и скорость: ближе `min_dist` → назад, дальше `max_dist` → вперёд на макс., между — пропорционально.

**Сборка RPM для Mecanum:**
```
w1 = −rotation_RPM + move_RPM   (переднее левое)
w2 = +rotation_RPM + move_RPM   (переднее правое)
w3 = +rotation_RPM + move_RPM   (заднее левое)
w4 = −rotation_RPM + move_RPM   (заднее правое)
```

### Логика потери цели

- `last_seen_time[track_id]` — время последнего обнаружения трека
- Если выбранный трек отсутствует дольше `TRACK_TIMEOUT` (2 с) → цель потеряна
- При потере начинается **поиск**: вращение со скоростью `SEARCH_ROTATION_SPEED = 0.3 об/с`
- Если поиск длится дольше `SEARCH_TIMEOUT` (5 с) — цель сбрасывается, движение останавливается

### Ручное управление (WASD)

Доступно только вне режима трекинга, обрабатывается через `pygame.key.get_pressed()`:

| Клавиша | Действие | Скорость |
|---|---|---|
| W / S | вперёд / назад | 0.8 м/с |
| A / D | стрейф влево / вправо | 0.5 м/с |

## Управление гимбалом

Угол наклона башни (pitch) задаётся в левой панели (0–35°, где 35° — максимально вверх).
Применяется кнопкой «Применить угол» через `RobotController.set_gimbal_pitch(angle)` — только при подключении и вне трекинга.

---

## Многопоточность

Основной поток — pygame mainloop (60 fps).

При подключении запускается `_camera_thread (daemon=True)`:
- Читает кадры с камеры (`read_cv2_image(strategy="newest")`)
- Кладёт сырой кадр в `_raw_frame` под `_raw_lock`

При запуске трекинга запускается `_track_thread (daemon=True)` (~30 fps):
- Копирует сырой кадр, запускает `tracker.process_frame()`
- Вызывает `_drive_from_tracking()` — управляет колёсами
- Кладёт аннотированный кадр в `_cur_frame` под `_frame_lock`

Остановка потоков — через `threading.Event`. Пересоздание `PersonTracker` при смене модели/устройства происходит в момент старта трекинга (в `_track_thread` до его запуска, в главном потоке).

## Интерфейс (v0.3)

Окно масштабируемое, запускается на весь экран. Три зоны:

### Левая панель (320 px, фиксированная)

Числовые поля (`SpinRow`) поддерживают два способа ввода:
- Кнопки **−** / **+** — шаговое изменение значения.
- **Клик по полю значения** — переход в режим редактирования (синяя рамка, мигающий курсор). Ввод с клавиатуры, **Enter** или клик вне поля — применить (значение зажимается в допустимый диапазон), **Escape** — отмена.

Параметры, доступные **во время трекинга**:
- Скорости вращения (макс./мин., об/с)
- Скорости вперёд (макс./мин., м/с)
- Скорости назад (макс./мин., м/с)

Параметры, доступные **до начала трекинга**:
- Дистанция следования (мин./макс., мм)

Параметры, доступные **только до подключения**:
- Выбор модели YOLO (yolo11n / yolo26n)
- Выбор устройства (CPU / GPU)

Параметры, доступные **при подключении, вне трекинга**:
- Угол гимбала + кнопка «Применить»

### Видеопанель (растягивается)

Кадры с камеры в BGR (OpenCV) → RGB (pygame) через `pygame.image.frombuffer`.
Кадр масштабируется с сохранением пропорций и центрируется.
Кликом выбирается цель. Оверлей: дистанция, время поиска.

### Правая панель (320 px, фиксированная)

- Индикаторы: связь, режим трекинга, ID цели, дистанция, модель, устройство
- Кнопки: Подключиться / Запуск трекинга / Остановить / Сброс цели / **Отключиться**
- Лог событий с переносом длинных строк по словам

```bash
source .venv/bin/activate
python main.py
```

Перед запуском робот должен быть включён, а компьютер подключён к его Wi-Fi точке доступа.
